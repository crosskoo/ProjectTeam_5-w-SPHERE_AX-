## 회의 내용

### 계획 및 요구 분석 발표 피드백
-13팀 : 다음 발표에서는 구현한 것을 기반으로 정리
-12팀 : 수행 배경 및 목표는 간단히 소개만, 다이어그램이 없음. 다음 시간에 추가로 그리기, 유즈케이스 다이어그램에서 외부 시스템도 actor 로 들어가야한다(박스형태로)
-11팀 : 구조, 동작 모델 다음 시간 발표에 추가. 아
-10팀 : task 가 명시적으로 적힌 간트차트(일정) 이 필요.
-9팀 : 실시간 X -> 최신 데이터로 용어 고치기, 성능 관련해 논문을 쓸 때는 비교군이 필요.
-8팀 : 다음 발표 시 변경되거나 상세화된 부분 위주로 발표(모델 설계 같은 경우 다 완료되었으므로)
-6팀 : 부가적인 기능들은 세세하게 안해도 됨.
-4팀 : 유스케이스 다이어 그램은 기능별로 따로 그리는 것이 아님.
-3팀 :  유스케이스는 잘했다. 백엔드 진행상황에 나와있는 것 처럼 전체 세부 내용 구조를 다음 발표에 넣으면 좋겠음.


### 숙지 사항
-향후 일정 - 간트 차트로 표현할 것 (현재 진행된 사항, 예정된 사항 등을 표시)
-다이어그램 정리해놓기 -> 보고서에 추가할 수 있도록
-발표는 중요한 기능(주요 기능 위주) (로그인 등 일반적인 내용들은 간략히)
-2차 중간발표 : 5/7 (수)
-구현 내용을 바탕으로 한 상세 설계, 진행 상황.
-논문 구성 : LMS 공지에서 확인, (저자에 멘토님 + 소속 기관 포함해야)
-논문 교수님께 먼저 보내드리고 피드백 받을 수 있도록

### 발표 후 회의
- 논문 주제 정하기 -> 추후 회의 때 정하기로 결정
- 멘토 회의는 중간고사 끝나고 일정 잡기로 결정
