## 회의 내용

### 계획 및 요구 분석 발표 피드백
-13팀 : 상세 다이어그램을 최종 보고서에 넣기.
-12팀 : 최종 보고서에는 프레임워크를 상세하게 설명할 수 있는 설계를 넣기
-10팀 : 상세 설계가 없음. 차별점이 무엇인지가 중요함. github url 을 입력하면 될 수 있도록 하면 차별점이 생길 것 같음. 최종 보고서에 상세 설계 넣기
-9팀 : 보고서에는 내부 스트럭쳐 및 상세 다이어그램 추가가 필요
-8팀 : 보고서 작성 시에는 상세 설계 부분을 클래스 다이어그램 등 동작, 흐름을 확인할 수 있는 다이어그램을 사용할 것.
-6팀 : 시퀀스 다이어그램엔 actor 의 구분이 중요하다.
-5팀 : 질문(실제 연동 계획은? -> 아직까지는 게획은 없고 모델에 집중), 질문(안개, 구름 같은 각 클래스를 구분하여 보낼 수 있는가 -> DB 스키마가 지원할 수 있도록 설계됨)
-4팀 : 질문(테스트 계획은?)
-3팀 : 테스트 계획은 어떻게 되는지 정해야할 것.
-1팀 : 추후 보고서 작성에는 각종 다이어그램이 필요. 진행상황은 기술 위주로 구체적으로 구현된 것을 보여주었으면 좋았을 듯.
